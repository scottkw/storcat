## Cross-Phase Integration: 17/18 connections wired

### Critical Integration Issue

**build-windows fan-in blocker**: The eSigner sign steps in `release.yml` `build-windows` job have no conditional `if:` guards. When `ES_USERNAME`, `ES_PASSWORD`, `CREDENTIAL_ID`, `ES_TOTP_SECRET` are absent (current state), the `SSLcom/esigner-codesign` action will hard-fail. Because the `release` job has `needs: [build-macos, build-windows, build-linux-amd64, build-linux-arm64]`, a failed `build-windows` blocks the entire release — no GitHub Release is created for ANY platform.

**Impact:** A `v*.*.*` tag push today will produce no release artifacts. macOS and Linux builds succeed but their artifacts are orphaned because the fan-in release job never runs.

**Fix options:**
1. Add `if: secrets.ES_USERNAME != ''` to the 3 Windows signing steps (Sign portable exe, Sign NSIS installer, Verify Authenticode signatures)
2. Obtain SSL.com OV certificate and store the 4 secrets now
3. Remove `build-windows` from the `release` job's `needs:` array temporarily

### Other Integration Findings

- **WinGet first submission** (LOW): `vedantmgoyal9/winget-releaser` requires `scottkw.StorCat` to already exist in `microsoft/winget-pkgs`. First submission is manual. Documented deferral.
- **update-tap.sh stale URL** (INFO): Manual fallback script has wrong DMG filename pattern (`StorCat-${VERSION}-universal.dmg` vs actual `StorCat-v${VERSION}-darwin-universal.dmg`). CI path unaffected.
