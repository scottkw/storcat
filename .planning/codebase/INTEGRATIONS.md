# External Integrations

**Analysis Date:** 2026-03-24

## APIs & External Services

**None.** StorCat is a fully offline desktop application. It does not call any external APIs, cloud services, or remote endpoints.

## Data Storage

**Databases:**
- None. All data is stored as JSON files on the local filesystem.

**File Storage:**
- Local filesystem only
- Catalog data: JSON files (`.json`) written to user-selected directories via `src/catalog-service.js`
- Catalog views: HTML files (`.html`) generated alongside JSON catalogs
- App preferences: `preferences.json` stored in Electron's `userData` directory (`app.getPath('userData')`)
- Window state: `window-state.json` stored in Electron's `userData` directory

**Caching:**
- None. Catalogs are read from disk on each operation.

## Electron IPC Interface

The app uses Electron's `contextBridge` + `ipcRenderer.invoke` pattern for all main-renderer communication. The preload script (`src/preload.js`) exposes `window.electronAPI` with typed interface at `src/renderer/types/electron.d.ts`.

**IPC Channels (all use invoke/handle pattern):**

| Channel | Direction | Purpose |
|---------|-----------|---------|
| `select-directory` | Renderer -> Main | Open native directory picker for catalog source |
| `select-search-directory` | Renderer -> Main | Open native directory picker for search location |
| `select-output-directory` | Renderer -> Main | Open native directory picker for copy destination |
| `create-catalog` | Renderer -> Main | Traverse directory tree, write JSON + HTML catalog files |
| `search-catalogs` | Renderer -> Main | Search all `.json` catalog files in a directory for a term |
| `load-catalog` | Renderer -> Main | Read and parse a single catalog JSON file |
| `get-catalog-files` | Renderer -> Main | List all `.json` files in a directory with metadata |
| `get-catalog-html-path` | Renderer -> Main | Check if HTML companion file exists for a catalog |
| `read-html-file` | Renderer -> Main | Read HTML file contents for in-app rendering |
| `set-window-persistence` | Renderer -> Main | Enable/disable window position persistence |
| `get-window-persistence` | Renderer -> Main | Get current window persistence setting |
| `open-external` | Renderer -> Main | Open URL in system default browser via `shell.openExternal` |

## File System Interactions

**Read Operations:**
- `fs.readdir` - List directory contents for catalog creation and catalog file discovery (`src/catalog-service.js`, `src/main.js`)
- `fs.stat` - Get file/directory metadata (size, modification time) (`src/catalog-service.js`, `src/main.js`)
- `fs.readFile` - Read catalog JSON/HTML files and preferences (`src/main.js`, `src/catalog-service.js`)
- `fs.access` - Check file existence (HTML companion files) (`src/main.js`)

**Write Operations:**
- `fs.writeFile` - Write catalog JSON, catalog HTML, window state, and preferences (`src/catalog-service.js`, `src/main.js`)
- `fs.copyFile` - Copy catalog files to secondary output directory (`src/catalog-service.js`)
- `fs.unlink` - Delete window state file when persistence is disabled (`src/main.js`)

**Directory Traversal:**
- Recursive traversal in `traverseDirectory()` (`src/catalog-service.js` line 49)
- Skips hidden files/directories (names starting with `.`)
- Sorts directories first, then files, both alphabetically
- Silently skips permission-denied entries

## Catalog Data Format

**JSON Catalog Structure:**
```json
{
  "type": "directory",
  "name": "./relative-path",
  "size": 12345,
  "contents": [
    { "type": "file", "name": "./relative-path/file.txt", "size": 100 },
    { "type": "directory", "name": "./relative-path/subdir", "size": 200, "contents": [...] }
  ]
}
```

**Compatibility:** The search function (`src/catalog-service.js` line 279-289) handles two JSON formats:
1. Object format (generated by StorCat): `{ "type": "directory", ... }`
2. Array format (generated by legacy `tree` command / `sdcat.sh`): `[{ "contents": [...] }]`

**HTML Catalog:**
- Mimics output of Unix `tree` command (v1.7.0 format)
- Generated by `generateTreeHTML()` in `src/catalog-service.js` line 111
- Tree-like display with `[size]` prefixes and Unicode box-drawing characters

## Legacy CLI Tool

**`sdcat/sdcat.sh` and `sdcat/sdcat_zsh.sh`:**
- Bash/Zsh scripts that wrap the Unix `tree` command
- Predecessor to the Electron app
- Creates catalogs using `tree -h --du -f -J` (JSON output)
- Generates HTML catalogs using `tree -h --du -f -H`
- Search functionality via grep on JSON files
- `examples/` directory contains sample catalog output (`sd01.json`, `sd01.html`)

## Network Protocols

**None.** The application operates entirely offline. The only network-like communication is:
- `file://` protocol for loading the built renderer HTML in production mode (`src/main.js` line 138)
- `http://localhost:5173` for connecting to Vite dev server in development mode (`src/main.js` line 115)
