<objective>
Sign, notarize, and staple every macOS DMG produced by CI release tags so Gatekeeper accepts them without prompting on macOS 15+.

Purpose: macOS 15+ blocks unsigned/unnotarized apps. Without this, users who download StorCat DMGs from GitHub Releases see a scary Gatekeeper dialog or cannot open the app at all.

Output: Updated release.yml with complete signing pipeline; entitlements.plist for Wails runtime.
</objective>

  <action>
Modify the `build-macos` job in `.github/workflows/release.yml`. The changes must maintain strict ordering:
`wails build` -> `codesign .app` -> `codesign --verify` -> `hdiutil create DMG` -> `notarytool submit` -> `stapler staple` -> `spctl --assess` -> `upload-artifact`. Keychain cleanup runs `if: always()` at the end.

**1. Add `environment: release` to the `build-macos` job** (per SIGN-04, Pitfall 7):
Insert `environment: release` immediately after `runs-on: macos-14` so the job can access signing secrets stored in the GitHub `release` environment.

**2. Add certificate import step** (per SIGN-04) — insert AFTER "Verify universal binary" step and BEFORE the existing "Create macOS DMG" step:

```yaml
    - name: Import Apple certificate
      uses: apple-actions/import-codesign-certs@b610f78488812c1e56b20e6df63ec42d833f2d14  # v6.0.0
      with:
        p12-file-base64: ${{ secrets.APPLE_CERTIFICATE }}
        p12-password: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
```

**3. Add codesign step** (per SIGN-01) — insert AFTER certificate import and BEFORE "Create macOS DMG":

```yaml
    - name: Sign .app bundle
      run: |
        /usr/bin/codesign \
          --sign "${{ secrets.APPLE_CERTIFICATE_NAME }}" \
          --options runtime \
          --entitlements build/darwin/entitlements.plist \
          --force \
          --timestamp \
          --verbose \
          build/bin/StorCat.app

    - name: Verify code signature
      run: codesign --verify --verbose build/bin/StorCat.app
```

Do NOT use `--deep` flag (not needed for StorCat's lean 3-file bundle, and unreliable for complex bundles).

**4. Keep existing "Create macOS tarball" and "Create macOS DMG" steps as-is** — they already come after wails build. The codesign step is now inserted before them, so the DMG will contain a signed .app.

**5. Add notarization step** (per SIGN-02) — insert AFTER "Create macOS DMG" and BEFORE "Upload macOS artifact":

```yaml
    - name: Notarize DMG
      run: |
        xcrun notarytool store-credentials "notarytool-profile" \
          --apple-id "${{ secrets.APPLE_ID }}" \
          --team-id "${{ secrets.APPLE_TEAM_ID }}" \
          --password "${{ secrets.APPLE_NOTARIZATION_PASSWORD }}"

        xcrun notarytool submit \
          "build/bin/StorCat-${{ steps.version.outputs.VERSION }}-darwin-universal.dmg" \
          --keychain-profile "notarytool-profile" \
          --wait \
          | tee /tmp/notary-output.log

        grep -q "status: Accepted" /tmp/notary-output.log || {
          echo "::error::Notarization failed"
          SUBMISSION_ID=$(grep "  id: " /tmp/notary-output.log | head -1 | awk '{print $2}')
          if [ -n "$SUBMISSION_ID" ]; then
            xcrun notarytool log "$SUBMISSION_ID" --keychain-profile "notarytool-profile"
          fi
          exit 1
        }
```

Note: Uses `secrets.APPLE_ID` (the Apple account email). This is a new 10th secret that the user must add to the `release` environment. The checkpoint in Task 3 handles this.

Per Pitfall 3: Must grep for "status: Accepted" because `notarytool submit --wait` exits 0 even on rejection.

**6. Add staple step** (per SIGN-03) — insert AFTER notarization:

```yaml
    - name: Staple notarization ticket
      run: |
        xcrun stapler staple \
          "build/bin/StorCat-${{ steps.version.outputs.VERSION }}-darwin-universal.dmg"
```

**7. Add Gatekeeper verification step** (per SIGN-06) — insert AFTER stapling:

```yaml
    - name: Verify Gatekeeper acceptance
      run: spctl --assess --type exec build/bin/StorCat.app || exit 1
```

Per Pitfall 5: This MUST run after notarization+stapling, not before. spctl checks Apple's notarization database.

**8. Add keychain cleanup step** (per SIGN-04) — insert as the LAST step in the job:

```yaml
    - name: Clean up keychain
      if: always()
      run: security delete-keychain signing_temp.keychain-db 2>/dev/null || true
```

`if: always()` ensures cleanup runs even if signing/notarization fails.
