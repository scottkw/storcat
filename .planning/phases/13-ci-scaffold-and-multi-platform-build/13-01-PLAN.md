Create `.github/workflows/release.yml` with the following exact structure. Every value below is concrete — implement verbatim.

**Trigger:**
```yaml
name: Release
on:
  push:
    tags: ['v*.*.*']
```

**Permissions (top-level):**
```yaml
permissions:
  contents: write
```

**Jobs (5 total, 4 build + 1 release):**

**Job 1: build-macos**
- `runs-on: macos-14` (CICD-02 — NOT macos-latest)
- Steps: checkout, setup-go (go-version-file: go.mod), setup-node (node-version: '20'), install Wails (`go install github.com/wailsapp/wails/v2/cmd/wails@v2.10.2` — pin to project version, NOT @latest), get version from tag (`echo "VERSION=${GITHUB_REF#refs/tags/}" >> $GITHUB_OUTPUT`), build macOS universal (`wails build -clean -platform darwin/universal`), verify universal binary (`lipo -info build/bin/StorCat.app/Contents/MacOS/StorCat`), create tar.gz (`cd build/bin && tar czf StorCat-${VERSION}-darwin-universal.tar.gz StorCat.app`), upload artifact (name: `StorCat-darwin-universal`, path: `build/bin/StorCat-*-darwin-universal.tar.gz`, retention-days: 1)

**Job 2: build-windows**
- `runs-on: windows-latest` (CICD-03 — NOT macos-latest like the buggy build.yml)
- Steps: checkout, setup-go, setup-node, install Wails, get version from tag (use `shell: bash`), build Windows (`wails build -clean -platform windows/amd64 -windowsconsole` — the `-windowsconsole` flag is REQUIRED per CICD-03), rename binary (`mv build/bin/StorCat.exe build/bin/StorCat-${VERSION}-windows-amd64.exe` using `shell: bash`), upload artifact (name: `StorCat-windows-amd64`, path: `build/bin/StorCat-*-windows-amd64.exe`, retention-days: 1)

**Job 3: build-linux-amd64**
- `runs-on: ubuntu-22.04` (CICD-04 — NOT ubuntu-latest)
- Steps: checkout, setup-go, setup-node, install Linux deps (`sudo apt-get update && sudo apt-get install -y libgtk-3-dev libwebkit2gtk-4.0-dev`), install Wails, get version from tag, build Linux amd64 (`wails build -clean -platform linux/amd64`), rename and tar (`cd build/bin && tar czf StorCat-${VERSION}-linux-amd64.tar.gz StorCat`), upload artifact (name: `StorCat-linux-amd64`, path: `build/bin/StorCat-*-linux-amd64.tar.gz`, retention-days: 1)

**Job 4: build-linux-arm64**
- `runs-on: ubuntu-22.04-arm` (CICD-04 — native ARM64 runner, no cross-compilation)
- Steps: identical to build-linux-amd64 except: platform is `linux/arm64`, artifact name is `StorCat-linux-arm64`, tar filename uses `linux-arm64`

**Job 5: release (fan-in)**
- `needs: [build-macos, build-windows, build-linux-amd64, build-linux-arm64]` (CICD-05)
- `runs-on: ubuntu-22.04`
- Steps: download all artifacts (`actions/download-artifact` with `path: artifacts/`), list artifacts for debugging (`ls -laR artifacts/`), create GitHub release using `softprops/action-gh-release` with `draft: true`, `generate_release_notes: true`, `files: artifacts/**/*`

**SHA Pinning (CICD-06) — use these EXACT commit SHAs with version comments:**
- `actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683` # v4.2.2
- `actions/setup-go@0aaccfd150d50ccaeb58ebd88d36e91967a5f35b` # v5.4.0
- `actions/setup-node@49933ea5288caeca8642d1e84afbd3f7d6820020` # v4.4.0
- `actions/upload-artifact@ea165f8d65b6e75b540449e92b4886f43607fa02` # v4.6.2
- `actions/download-artifact@95815c38cf2ff2164869cbab79da8d1f422bc89e` # v4.2.1
- `softprops/action-gh-release@153bb8e04406b158c6c84fc1615b65b24149a1fe` # v2.6.1

Every `uses:` line MUST use the full 40-char SHA followed by a `# vX.Y.Z` comment. No tag references like `@v4` anywhere.
