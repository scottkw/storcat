  <name>Task 1: Create Linux desktop entry file</name>
  <files>build/linux/storcat.desktop</files>
  <action>
Create `build/linux/storcat.desktop` with the following exact content (freedesktop.org desktop entry spec):

```ini
[Desktop Entry]
Type=Application
Name=StorCat
Comment=Storage Media Cataloging Tool
Exec=StorCat
Icon=appicon
Categories=Utility;FileManager;
Terminal=false
```

This file is required by both linuxdeploy (AppImage, PKG-03) and dpkg-deb (.deb, PKG-04). The `Icon=appicon` references the icon name without extension -- linuxdeploy resolves this from the `--icon-file` flag, and the .deb build copies `build/appicon.png` to `/usr/share/pixmaps/storcat.png`.

Do NOT add a trailing newline after `Terminal=false` beyond the standard single newline.

  </verify>

**build-macos job (PKG-01):** Add two steps after "Create macOS tarball":

1. "Install create-dmg" step:
   ```yaml
   - name: Install create-dmg
     run: npm install --global create-dmg
   ```

2. "Create macOS DMG" step:
   ```yaml
   - name: Create macOS DMG
     run: |
       cd build/bin
       create-dmg \
         --overwrite \
         --dmg-title "StorCat" \
         --no-code-sign \
         "StorCat-${{ steps.version.outputs.VERSION }}-darwin-universal.dmg" \
         "StorCat.app"
   ```

**build-windows job (PKG-02):**

1. Change `runs-on: windows-latest` to `runs-on: windows-2022`

2. Replace the existing "Build Windows" step with one that includes `-nsis`:
   ```yaml
   - name: Build Windows with NSIS installer
     run: wails build -clean -platform windows/amd64 -windowsconsole -nsis
   ```
   Wails auto-generates `build/windows/installer/` if it does not exist, then calls `makensis` to produce the installer.

3. Keep the existing "Rename Windows binary" step as-is (it renames the standalone exe).

4. Add a step to rename the NSIS installer for versioned naming:


**build-linux-amd64 job (PKG-03 + PKG-04 amd64):** Add steps after "Create Linux amd64 tarball":

1. "Install AppImage tools" step:
   ```yaml
   - name: Install AppImage tools
     run: |
       sudo apt-get install -y libfuse2
       wget -q -O linuxdeploy https://github.com/linuxdeploy/linuxdeploy/releases/download/continuous/linuxdeploy-x86_64.AppImage
       wget -q -O linuxdeploy-plugin-appimage https://github.com/linuxdeploy/linuxdeploy-plugin-appimage/releases/download/continuous/linuxdeploy-plugin-appimage-x86_64.AppImage
       wget -q -O linuxdeploy-plugin-gtk.sh https://raw.githubusercontent.com/linuxdeploy/linuxdeploy-plugin-gtk/master/linuxdeploy-plugin-gtk.sh
       chmod +x linuxdeploy linuxdeploy-plugin-appimage linuxdeploy-plugin-gtk.sh
   ```

2. "Create AppImage" step:
   ```yaml
   - name: Create AppImage
     run: |
       export ARCH=x86_64
       export OUTPUT=StorCat-x86_64.AppImage
       ./linuxdeploy \
         --appdir AppDir \
         --executable build/bin/StorCat \
         --icon-file build/appicon.png \
         --desktop-file build/linux/storcat.desktop \
         --plugin gtk \
         --output appimage
       mv StorCat-x86_64.AppImage \
         build/bin/StorCat-${{ steps.version.outputs.VERSION }}-linux-x86_64.AppImage
   ```
   Note: linuxdeploy outputs to current directory, not build/bin -- the mv is required (pitfall #7).

3. "Create .deb package" step (amd64):
   ```yaml
   - name: Create .deb package
     run: |
       ARCH=$(dpkg --print-architecture)
       VERSION_CLEAN="${{ steps.version.outputs.VERSION }}"
       VERSION_CLEAN="${VERSION_CLEAN#v}"
       PKG_DIR="deb-pkg"
       mkdir -p "$PKG_DIR/DEBIAN" "$PKG_DIR/usr/bin" \
                "$PKG_DIR/usr/share/applications" "$PKG_DIR/usr/share/pixmaps"
       cp build/bin/StorCat "$PKG_DIR/usr/bin/StorCat"
       cp build/linux/storcat.desktop "$PKG_DIR/usr/share/applications/storcat.desktop"
       cp build/appicon.png "$PKG_DIR/usr/share/pixmaps/storcat.png"
       chmod 755 "$PKG_DIR/usr/bin/StorCat"
       printf 'Package: storcat\nVersion: %s\nArchitecture: %s\nMaintainer: Ken Scott <kenscott@gmail.com>\nDescription: Storage Media Cataloging Tool\n StorCat creates, browses, and searches directory catalogs.\n Generates JSON and HTML representations of directory trees.\nDepends: libgtk-3-0, libwebkit2gtk-4.0-37\nSection: utils\nPriority: optional\n' \
         "$VERSION_CLEAN" "$ARCH" > "$PKG_DIR/DEBIAN/control"
       dpkg-deb --root-owner-group --build "$PKG_DIR" \
         "build/bin/storcat_${VERSION_CLEAN}_${ARCH}.deb"
   ```
   Note: Version string strips leading 'v' (pitfall #5). Architecture auto-detected from runner.

4. Update upload-artifact path to include all three:
   ```yaml
   path: |
     build/bin/StorCat-*-linux-amd64.tar.gz
     build/bin/StorCat-*-linux-x86_64.AppImage
     build/bin/storcat_*_amd64.deb
   ```
