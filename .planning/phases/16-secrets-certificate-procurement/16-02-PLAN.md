Per CRED-01: Verify the existing Developer ID Application certificate on the local keychain.

1. Run identity check:
```bash
security find-identity -v -p codesigning
```
Expected output includes: `Developer ID Application: Ken Scott (S2K7P43927)`

2. Check certificate expiry:
```bash
security find-certificate -c "Developer ID Application: Ken Scott" -p | openssl x509 -noout -dates
```
Expected: `notAfter=Feb  1 22:12:15 2027 GMT` (valid for ~11 more months)

3. If the certificate is NOT found or expired, STOP and report to user -- they must renew via developer.apple.com before proceeding.

4. Record the exact identity string for use as `APPLE_CERTIFICATE_NAME`.
  </action>

CHECKPOINT: Human action required. Claude cannot perform these steps because they require Keychain Access GUI interaction and Apple ID web login.

**What was built:** Certificate verified as present on local keychain (Task 1). Now the user needs to export it and generate an Apple app-specific password.

**Instructions for user:**

Action A: Export .p12 from Keychain Access
1. Open Keychain Access.app
2. Select "login" keychain, then "My Certificates" category in the left sidebar
3. Find and expand "Developer ID Application: Ken Scott (S2K7P43927)" to see the private key beneath it
4. Click the certificate row to select it
5. Cmd+click the private key row beneath it (select BOTH items -- this is critical per Pitfall 1: exporting only the cert without the key causes codesign to fail with "no private key found")
6. Right-click the selection -> "Export 2 Items..."
7. Save as `storcat-dev-id.p12` in a temporary location (e.g., Desktop)
8. Enter a strong password when prompted -- RECORD THIS PASSWORD (it becomes APPLE_CERTIFICATE_PASSWORD)

Action B: Generate app-specific password for notarization
1. Go to https://appleid.apple.com
2. Sign in with your Apple ID
3. Navigate to: Sign-In and Security -> App-Specific Passwords
4. Click "+" to generate a new password
5. Name it "StorCat CI" for traceability (per Pitfall 2: do not use your main Apple ID password)
6. RECORD THE GENERATED PASSWORD (it becomes APPLE_NOTARIZATION_PASSWORD)

**Resume signal:** Provide the .p12 file path, the .p12 password, and the app-specific password. Type "done" with these values, or describe any issues.

Per CRED-02 and CRED-05 (Apple portion): Store all 5 Apple secrets in the GitHub `release` environment.

Using the .p12 file path, .p12 password, and app-specific password provided by the user in Task 2:

1. Base64-encode the .p12 and store as APPLE_CERTIFICATE:
```bash
gh secret set APPLE_CERTIFICATE --env release --repo scottkw/storcat \
  --body "$(base64 -i /path/to/storcat-dev-id.p12)"
```

2. Store the .p12 export password:
```bash
gh secret set APPLE_CERTIFICATE_PASSWORD --env release --repo scottkw/storcat \
  --body "THE_P12_PASSWORD"
```

3. Store the certificate identity name:
```bash
gh secret set APPLE_CERTIFICATE_NAME --env release --repo scottkw/storcat \
  --body "Developer ID Application: Ken Scott (S2K7P43927)"
```

4. Store the app-specific password for notarization:
```bash
gh secret set APPLE_NOTARIZATION_PASSWORD --env release --repo scottkw/storcat \
  --body "THE_APP_SPECIFIC_PASSWORD"
```

5. Store the Team ID:
```bash
gh secret set APPLE_TEAM_ID --env release --repo scottkw/storcat \
  --body "S2K7P43927"
```

6. Verify all 5 secrets exist:
```bash
gh secret list --env release --repo scottkw/storcat
```

7. After secrets are stored, securely delete the .p12 file:
```bash
rm -P /path/to/storcat-dev-id.p12
```

IMPORTANT: Use the exact secret names above -- Phase 17 will reference these names in workflow YAML. Do NOT use alternative names.
