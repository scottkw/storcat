CHECKPOINT: Decision required from user.

**Decision:** Which Windows code signing service to use for Authenticode signing in CI.

Per CRED-03: A Windows OV code signing certificate must be obtained with RSA (not ECDSA) confirmed before purchase. Since June 2023, all new OV certificates are non-exportable (keys on cloud HSMs). Traditional .pfx export is no longer possible for new certificates.

**Option A: SSL.com eSigner OV (Recommended)**
Pros:
- $20/month ($240/year), 20 signings per month (sufficient for CI releases)
- Full CI automation via `SSLcom/esigner-codesign` GitHub Action
- No hardware token required
- RSA by default for OV tier
- 4 API secrets: ES_USERNAME, ES_PASSWORD, CREDENTIAL_ID, ES_TOTP_SECRET
- Available to individual developers worldwide

Cons:
- Monthly cost ($20/month)
- SmartScreen reputation builds over time (not instant, same as any OV cert)
- 20 signing limit per month on base tier

**Option B: Azure Artifact Signing (formerly Trusted Signing)**
Pros:
- $9.99/month
- Instant SmartScreen reputation (Microsoft-backed)

Cons:
- RESTRICTED: As of April 2025, only available to US/Canadian businesses with 3+ years of history
- Individual developers are currently ineligible

**Option C: Defer Windows signing to a future milestone**
Pros:
- No cost now
- Can still complete macOS signing (Phases 16-17, 19) without Windows cert

Cons:
- Phase 18 (Windows Authenticode) and Phase 20 (WinGet CLI PATH) cannot proceed
- Windows users continue seeing SmartScreen warnings

**Resume signal:** Select option-a, option-b, or option-c.

**Instructions for user:**

Action A: Create SSL.com account and purchase OV code signing cert
1. Go to https://www.ssl.com
2. Create an account (or log in if you have one)
3. Purchase an OV Code Signing certificate
4. IMPORTANT: Confirm the certificate uses RSA algorithm (not ECDSA) -- per research Pitfall 3, ECDSA has poor SmartScreen reputation history with signtool
5. Complete identity verification as required by SSL.com

Action B: Enroll in eSigner
1. After certificate is issued, go to SSL.com dashboard
2. Navigate to your certificate -> eSigner enrollment
3. Follow the eSigner enrollment wizard
4. CRITICAL (Pitfall 4): When the TOTP QR code is displayed, extract the `secret=XXXXXX` value from the `otpauth://` URI. This is shown ONLY ONCE. Save it immediately as ES_TOTP_SECRET.
5. Record your Credential ID from the SSL.com dashboard

When enrollment is complete, provide these 4 values:
- SSL.com username (email)
- SSL.com password
- Credential ID
- TOTP secret (the base32 seed from the QR code URI)
