    4. Write `packaging/homebrew/storcat.rb.template` with the following EXACT content:

    ```ruby
    cask "storcat" do
      version "{{VERSION}}"
      sha256 "{{SHA256}}"

      url "https://github.com/scottkw/storcat/releases/download/#{version}/StorCat-#{version}-universal.dmg"

      name "StorCat"
      desc "Directory Catalog Manager - Create, search, and browse directory catalogs"
      homepage "https://github.com/scottkw/storcat"

      livecheck do
        url :url
        strategy :github_latest
      end

      app "StorCat.app"

      zap trash: [
        "~/Library/Application Support/StorCat",
        "~/Library/Preferences/com.kenscott.storcat.plist",
        "~/Library/Saved Application State/com.kenscott.storcat.savedState",
        "~/Library/Caches/com.kenscott.storcat",
      ]
    end
    ```

    CRITICAL: The `#{version}` in the `url` line is Homebrew Ruby string interpolation — it must remain as literal `#{version}`, NOT be replaced with `{{VERSION}}`. Only the `version` and `sha256` field values use the `{{...}}` placeholder syntax. The update script will substitute `{{VERSION}}` and `{{SHA256}}` with real values, and the resulting cask file will use Ruby's `#{version}` to interpolate the version into the URL at brew install time.

    5. Copy `update-tap.sh` from `/tmp/homebrew-storcat-migration/update-tap.sh` to `packaging/homebrew/update-tap.sh`

    6. Ensure execute permission: `chmod +x packaging/homebrew/update-tap.sh`

    7. Clean up: `rm -rf /tmp/homebrew-storcat-migration`

    NOTE: The copied `update-tap.sh` script currently assumes it runs from within the `homebrew-storcat` repo root (it uses `CASK_FILE="Casks/storcat.rb"` relative to CWD and does `git add`/`git commit`/`git push`). This cross-repo limitation is documented and will be addressed in Phase 15. For Phase 12, copy it as-is — it serves as a reference for the Phase 15 automation work.
