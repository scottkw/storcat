## Phase Scope Note

Phase 4 is a **Go method fix + TypeScript shim correction** phase. It does not introduce new screens, new components, or new routes. Every UI change is confined to:

1. `app.go` тАФ `GetCatalogHtmlPath` implementation (adds `os.Stat` existence check)
2. `frontend/src/services/wailsAPI.ts` тАФ 7 wrapper methods return `{success, ...}` envelopes
3. `frontend/src/components/CatalogModal.tsx` тАФ consumer updated to use new envelope shapes

No new visual elements are added. The design contract governs the **error and loading states** that become visible once the wrappers return proper envelopes.

## Interaction States

These states become correctly visible after the envelope fixes. Each must be implemented exactly as specified.

### CatalogModal тАФ HTML Load Flow

| State | Trigger | Visual | Copy |
|-------|---------|--------|------|
| Loading | `visible=true` and `catalogPath` set | Centered `div` with text "Loading..." at full modal height тАФ existing implementation, no change | "Loading..." |
| Success | `result.success === true` | `iframe` with `srcDoc={result.content}` fills modal body тАФ existing implementation, no change | (no user-visible copy) |
| HTML not found | `result.success === false` from `getCatalogHtmlPath` | `message.error(...)` toast тАФ existing Ant Design notification | "HTML file not found for this catalog" |
| Read failed | `result.success === false` from `readHtmlFile` | `message.error(...)` toast | "Failed to read HTML file" |
| Unexpected error | `catch` block | `message.error(...)` toast | "Failed to load catalog HTML" |

### Directory Picker тАФ selectDirectory (and aliases)

| State | Trigger | Visual |
|-------|---------|--------|
| Success | User selects directory | Caller uses `result.path` to update form field тАФ existing behavior, no new UI |
| Cancelled / error | User cancels dialog or error | `result.success === false` тАФ caller checks and takes no action (existing behavior: no path update) |

### Config Load тАФ getConfig

| State | Trigger | Visual |
|-------|---------|--------|
| Success | App startup | `result.config` consumed by App.tsx to set theme/sidebar тАФ existing behavior |
| Error | `result.success === false` | App falls back to defaults тАФ no user-visible error message for config load failure |

### Window Persistence Toggle тАФ getWindowPersistence

| State | Trigger | Visual |
|-------|---------|--------|
| Enabled | `result.success && result.enabled === true` | Toggle shows ON state in settings modal |
| Disabled | `result.enabled === false` | Toggle shows OFF state |
| Error fallback | `result.success === false` | `result.enabled` defaults to `true` тАФ toggle shows ON (safe default) |

| Wrapper | Success Shape | Failure Shape |
|---------|--------------|---------------|
| `selectDirectory` | `{success: true, path: string}` | `{success: false, error: string}` |

| `selectOutputDirectory` | `{success: true, path: string}` | `{success: false, error: string}` |
