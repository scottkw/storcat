### Integration Findings

1. **CATL-02 result display gap (Info):** `wailsAPI.createCatalog` returns all 6 fields from `CreateCatalogResult` but `CreateCatalogTab` only reads `result.success`. The Electron v1 equivalent displayed output paths. This is a presentation gap, not an API gap — the data flows correctly through the shim layer. Deferred to ENH-03.

2. **CATL-01 orphaned consumer (Info):** `loadCatalog` wailsAPI wrapper exists and is fully wired from Go to TypeScript, but no React component currently calls it. The CatalogModal uses `getCatalogHtmlPath` + `readHtmlFile` for HTML preview instead. The JSON viewer path is available but unused.

3. **PLAT-02 documentation discrepancy (Info):** Phase 6 SUMMARY documents ldflags injection but implementation uses `go:embed wails.json` to read `productVersion`. Build flag `-X main.Version=...` would have no effect. Version display works correctly via wails.json.

4. **Latent nil pointer (Info):** `GetWindowPersistence` error path could panic if `config.NewManager()` fails — requires both `UserConfigDir()` and `UserHomeDir()` to fail simultaneously. Low probability, no production risk.
