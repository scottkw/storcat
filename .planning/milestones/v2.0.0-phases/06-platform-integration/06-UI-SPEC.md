## Interaction Contract

### PLAT-01 — Header Drag Region

| Property | Value |
|----------|-------|
| Target element | `AntHeader` in `Header.tsx` |
| CSS property to add | `'--wails-draggable': 'drag'` |
| TypeScript cast | `as React.CSSProperties & { '--wails-draggable'?: string }` |
| Interactive children needing no-drag override | None — header contains only icon + title text (non-interactive) |
| Visual change | None |
| Platform scope | macOS (primary). Property is inert on other platforms in Wails v2 framed mode. |
| Existing stub to retain | `WebkitAppRegion?: string` in existing cast — retain harmlessly or clean up; either is acceptable |

### PLAT-02 — Dynamic Version String

| Property | Value |
|----------|-------|
| Target element | `<Text strong>` in `CreateCatalogContent` (line 248–250 of `CreateCatalogTab.tsx`) |
| State variable | `const [appVersion, setAppVersion] = useState('...')` |
| Mount effect | `useEffect` calling `window.electronAPI?.getVersion?.()` once on mount |
| Success path | `result.success === true` → `setAppVersion(result.version)` |
| Failure path | `result.success === false` → `setAppVersion(result.version)` where version is `'dev'` from wailsAPI fallback |
| Release notes link | Uses `appVersion` state in URL: `https://github.com/scottkw/storcat/releases/tag/${appVersion}` |
| Loading flash | `"..."` briefly visible before first render cycle resolves; acceptable — no spinner required |
| New wailsAPI method | `getVersion: async () => { success: true, version: string } | { success: false, version: string }` |
