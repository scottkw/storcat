# Phase 1: Data Models + Catalog Service - Research

**Researched:** 2026-03-24
**Domain:** Go data models, JSON serialization, filesystem traversal, HTML generation
**Confidence:** HIGH

## Summary

Phase 1 fixes a set of concrete, auditable divergences between the current Go implementation and the Electron v1.2.3 reference. Every bug has been traced to a specific line in the codebase by reading both the Electron source (from git history, commit `9c38abb1`) and the current Go code. No speculative research is needed — the delta is fully known.

The Electron v1 format is:
- JSON: array-wrapped `[{...}]` (from the legacy bash script `sdcat.sh`)
- The Go v2 requirement (DATA-01) mandates **bare object `{...}`** — this is a deliberate format upgrade for Electron→Wails migration, not a regression. The search/browse code already reads both formats.

Six of the eight requirements require edits to existing Go files. No new packages are needed except `djherbis/times` for macOS birth time (DATA-05). The `CreateCatalog` signature change (CATL-02) is the most invasive change, as it affects `app.go`, `internal/catalog/service.go`, and the Wails-generated TypeScript bindings.

**Primary recommendation:** Fix bugs in dependency order — models first (DATA-01–05), then service signature (CATL-02), then traversal (CATL-03), then HTML output (CATL-04). Each fix is small and independently verifiable.

### Pattern 1: JSON Tag Fix for DATA-01 and DATA-02
**What:** Remove array wrapping in `writeJSONFile`; change `omitempty` to always emit `[]`.
**When to use:** Any time Contents must be present in JSON even when empty.
**Example:**
```go
// pkg/models/catalog.go — before
Contents []*CatalogItem `json:"contents,omitempty"`

// After
Contents []*CatalogItem `json:"contents"`
```
```go
// internal/catalog/service.go — before (writeJSONFile)
data := []*models.CatalogItem{catalog}
jsonBytes, err := json.Marshal(data)

// After
jsonBytes, err := json.Marshal(catalog)  // bare object
```

**CRITICAL:** The `traverseDirectory` function already initializes `Contents: []*models.CatalogItem{}` for empty directories. With `omitempty` removed, `nil` would serialize as `null`. The existing empty-slice initialization on the error branch (line 101-106) is correct, but the non-error branch that uses `append` must also be guarded: if `contents` is `nil` after the loop (no non-hidden entries), assign an empty slice before returning.

### Pattern 2: Result Struct for CATL-02
**What:** Define a `CreateCatalogResult` struct in `pkg/models/`, change `CreateCatalog` to return `(*CreateCatalogResult, error)`.
**When to use:** Any service method that must return data to the frontend.
**Example:**
```go
// pkg/models/catalog.go — new struct
type CreateCatalogResult struct {
    JsonPath     string `json:"jsonPath"`
    HtmlPath     string `json:"htmlPath"`
    FileCount    int    `json:"fileCount"`
    TotalSize    int64  `json:"totalSize"`
    CopyJsonPath string `json:"copyJsonPath,omitempty"`
    CopyHtmlPath string `json:"copyHtmlPath,omitempty"`
}
```
```go
// app.go — after
func (a *App) CreateCatalog(title string, directoryPath string, outputName string, copyToDirectory string) (*models.CreateCatalogResult, error) {
    ...
    return a.catalogService.CreateCatalog(title, absPath, outputName, copyToDirectory, progressCallback)
}
```

After changing `app.go`, run `wails generate module` from the project root to regenerate `frontend/wailsjs/go/main/App.d.ts` and `frontend/wailsjs/go/models.ts`.

### Pattern 3: Symlink Following for CATL-03
**What:** Replace `os.Lstat` with `os.Stat` in `traverseDirectory`.
**When to use:** When symlink targets should be traversed, not the symlink node itself.
**Example:**
```go
// internal/catalog/service.go — before
info, err := os.Lstat(dirPath)

// After
info, err := os.Stat(dirPath)
```
**Note:** Electron's `fs.stat` follows symlinks. `os.Lstat` reports the symlink itself (type `ModeSymlink`, not `IsRegular` or `IsDir`), causing symlinks to fall through to the `fmt.Errorf("unsupported file type")` path and be silently skipped. Changing to `os.Stat` makes symlinked files appear as regular files and symlinked directories get traversed recursively.

### Pattern 4: HTML Root Node Fix for CATL-04
**What:** The root node (where `item.Name == "./"`) currently outputs `./<br>` without a connector or size bracket. Electron's `generateTreeStructure(catalog)` is called with `isLast=true`, so the root node receives `└──` connector and a size bracket.
**When to use:** Always — root must match Electron's tree appearance.

Current Go code (lines 236-245 in `service.go`):
```go
if item.Name == "./" {
    result.WriteString("./<br>\n")  // wrong — no connector, no size
    ...
```

Electron equivalent:
```js
const generateTreeStructure = (item, isLast = true, prefix = '') => {
    let html = '';
    const connector = isLast ? '└── ' : '├── ';
    const sizeDisplay = formatBytesForDisplay(item.size);
    const itemName = path.basename(item.name);

    if (item.type === 'directory') {
        html += `${prefix}${connector}${sizeDisplay}&nbsp;&nbsp;${escapeHtml(itemName)}<br>\n`;
```

So for the root, Electron outputs: `└── [size]&nbsp;&nbsp;./<br>` (prefix is `""`, connector is `└──`, itemName is `basename("./") = "."`)

Fix: remove the special-case `if item.Name == "./"` block and let the root node follow the same path as all other nodes. The prefix starts at `""` and the caller passes `isLast=true` for the root.

### Pattern 5: CatalogMetadata Fields for DATA-03, DATA-04, DATA-05
**What:** Add `Size`, change `Created`/`Modified` to `time.Time` or keep as string with correct format.

Current `CatalogMetadata`:
```go
Created  string `json:"created"`    // "2006-01-02 15:04:05" — not Date-parseable in JS
Modified string `json:"modified"`   // "2006-01-02T15:04:05Z07:00" — IS Date-parseable
```

The frontend `BrowseCatalogsTab.tsx` line 150 does: `new Date(value).toLocaleString()` on the `modified` field. The current Go format `"2006-01-02T15:04:05Z07:00"` (RFC3339) IS Date-compatible. But `Created` uses `"2006-01-02 15:04:05"` which is NOT a valid JS Date string.

For DATA-04: `Modified` format is already correct (RFC3339 / ISO 8601). No change needed for `Modified`.
For DATA-03: Add `Size int64` field to `CatalogMetadata` and populate it from `info.Size()`.
For DATA-05: `Created` must use `BirthTime()` from `djherbis/times` on macOS/Windows; fall back to `ModTime()` on platforms where birth time is unavailable.

```go
// internal/catalog/service.go — BrowseCatalogs
import "gopkg.in/djherbis/times.v1"

t, err := times.Stat(filePath)
var createdTime time.Time
if err == nil && t.HasBirthTime() {
    createdTime = t.BirthTime()
} else {
    createdTime = info.ModTime()  // fallback
}

catalogs = append(catalogs, &models.CatalogMetadata{
    ...
    Size:     info.Size(),
    Created:  createdTime.Format(time.RFC3339),
    Modified: info.ModTime().Format(time.RFC3339),
    ...
})
```

### Anti-Patterns to Avoid
- **Keeping `omitempty` on `Contents`:** With `omitempty`, a `nil` slice serializes as absent and an empty slice `[]` serializes correctly. The fix is removing `omitempty` AND ensuring `Contents` is always initialized to `[]*CatalogItem{}` for directories (never `nil`).
- **Using `os.Lstat` for traversal:** This silently drops all symlinks. The fix is `os.Stat`.
- **Hardcoding `Created` as formatted string with wrong layout:** `"2006-01-02 15:04:05"` is not a JS `Date`-compatible string. Use RFC3339.
- **Calling `wails generate module` manually after struct changes:** Must be done, or the TypeScript bindings will be stale and `CreateCatalogResult` fields won't be visible to the frontend.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| File birth time on macOS | syscall + reflect + `Sys()` type assertion | `github.com/djherbis/times` | Cross-platform; handles macOS `Birthtimespec`, Windows `CreationTime`, Linux fallback; already a Go community standard |
| JSON serialization | custom marshaler | `encoding/json` with correct struct tags | Standard library is sufficient; the bugs are all tag-level fixes |
| HTML escaping | custom replace | `html.EscapeString` (already used) | Already correct in existing code |

**Key insight:** Every requirement in this phase is a surgical fix — wrong tag, wrong function call, wrong format string, missing struct field. No new abstractions needed.

## Common Pitfalls

### Pitfall 1: `nil` vs `[]` for Empty Contents
**What goes wrong:** Removing `omitempty` without ensuring `Contents` is initialized to `[]` causes `null` in JSON output instead of `[]`.
**Why it happens:** In Go, `var contents []*CatalogItem` starts as `nil`. If the directory has only hidden files (all skipped), `contents` is never appended to and stays `nil`. `json.Marshal(nil slice)` produces `null`.
**How to avoid:** After the entry loop, before building the return struct, do:
```go
if contents == nil {
    contents = []*models.CatalogItem{}
}
```
**Warning signs:** `null` appearing in JSON for empty directories.

### Pitfall 2: Wails Bindings Stale After Struct Changes
**What goes wrong:** TypeScript sees the old `CreateCatalog` signature (`Promise<void>`) even after Go returns `*CreateCatalogResult`. Frontend calls work but the result is `undefined`.
**Why it happens:** Wails generates TypeScript bindings at build time. If `wails generate module` is not run after changing Go method signatures, the old `.d.ts` files remain.
**How to avoid:** Run `wails generate module` from the project root after any change to `app.go` method signatures.
**Warning signs:** TypeScript type errors on `result.jsonPath` etc., or `undefined` result in `wailsAPI.createCatalog`.

### Pitfall 3: Symlink Cycles with `os.Stat`
**What goes wrong:** A symlink cycle (`a -> b -> a`) causes infinite recursion with `os.Stat`.
**Why it happens:** `os.Stat` follows symlinks, so a symlinked directory looks identical to a real directory.
**How to avoid:** Electron's code also uses `fs.stat` (not `fs.lstat`) and has no cycle protection — this is acceptable for the parity goal. For now, match Electron's behavior exactly. Cycle protection is a v2 enhancement.
**Warning signs:** Stack overflow or very long traversal on directories with circular symlinks.

### Pitfall 4: Output Path Bug in `CreateCatalog`
**What goes wrong:** Current code writes `jsonPath = filepath.Join(directoryPath, outputRoot+".json")` — output goes INTO the cataloged directory. This matches Electron's behavior (`path.join(directoryPath, ...)`) but may surprise users.
**Why it happens:** The Electron IPC handler passes `directoryPath` as the output location. CATL-02 doesn't change this behavior; it only adds metadata to the return value.
**How to avoid:** When implementing CATL-02, populate `jsonPath` and `htmlPath` from the paths actually written, not recalculated.

### Pitfall 5: `djherbis/times` Import Path
**What goes wrong:** Using `github.com/djherbis/times` (v2 module path) vs `gopkg.in/djherbis/times.v1`.
**Why it happens:** The library has two import paths. The pkg.go.dev canonical path is `github.com/djherbis/times`.
**How to avoid:** Use `go get github.com/djherbis/times` and import `"github.com/djherbis/times"`.

## Open Questions

1. **`wails generate module` regeneration step**
   - What we know: After `app.go` method signature changes, `wails generate module` must be run to update `frontend/wailsjs/go/main/App.d.ts` and `frontend/wailsjs/go/models.ts`.
   - What's unclear: Whether this needs to be a separate plan step or can be inline with the signature change.
   - Recommendation: Include as an explicit verification step in the CATL-02 plan.

2. **`filename` vs `name` field semantics in `CatalogMetadata`**
   - What we know: Both `Name` and `Filename` fields exist with identical values (both set to `entry.Name()`). The Electron v1 returns `name: path.basename(filePath, '.json')` (without extension) and no `filename` key.
   - What's unclear: Whether both fields are consumed by the frontend or if one is redundant.
   - Recommendation: Leave both as-is during Phase 1. Cleanup is out of scope.

3. **`Created` field in Electron v1**
   - What we know: Electron v1's `get-catalog-files` does NOT return a `created` field — only `modified`, `size`, `path`, `name`, `title`, `hasHtml`.
   - What's unclear: DATA-05 says `created` uses actual creation time. The Go code has a `Created` string field. This is a Go addition not present in Electron v1.
   - Recommendation: Implement DATA-05 as specified (use birth time where available), but note the field is a Go addition. The frontend `BrowseCatalogsTab` does not render `created` in its column definitions, so this is a data-correctness fix with no current UI impact.
