# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Data Models + Catalog Service.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Scope Note

Phase 1 is a pure backend Go phase. All 8 requirements (DATA-01 through DATA-05,
CATL-02 through CATL-04) target Go structs, JSON serialization, filesystem traversal,
and HTML generation. No new React components are introduced. No existing components
are modified in this phase.

The design contract below documents:
1. The existing design system the backend output must conform to
2. The data shapes the frontend already expects (which the Go fixes deliver)
3. The copywriting for the two states affected by corrected data (Create success, empty Browse)

This contract is the source of truth for phases 5 and beyond when the frontend shim
and UI layers consume the fixed backend output.

## Data Shape Contract

This section is specific to Phase 1. It documents the exact JSON shapes the frontend
already expects and that the Go fixes must deliver. This is the observable contract
between backend output and frontend consumption.

### Catalog JSON (DATA-01, DATA-02)

Expected shape after Phase 1 fix:
```json
{
  "type": "directory",
  "name": "./",
  "size": 12345,
  "contents": []
}
```

- Bare object `{...}` — not array-wrapped `[{...}]`
- `contents` always present as `[]` for empty directories — never `null` or absent
- Frontend `.map()` calls on `contents` must not receive `null` (causes TypeError)

### CreateCatalog Result (CATL-02)

Expected shape returned to frontend after Phase 1 fix:
```json
{
  "jsonPath": "/path/to/output.json",
  "htmlPath": "/path/to/output.html",
  "fileCount": 42,
  "totalSize": 123456,
  "copyJsonPath": "/path/to/copy.json",
  "copyHtmlPath": "/path/to/copy.html"
}
```

- `copyJsonPath` and `copyHtmlPath` are omitted when no copy directory was specified
- `fileCount` and `totalSize` are displayed to the user in the Create tab post-creation message

### Browse Catalog Metadata (DATA-03, DATA-04, DATA-05)

Expected shape for each catalog entry in BrowseCatalogsTab:
```json
{
  "path": "/absolute/path/to/catalog.json",
  "name": "catalog-name",
  "title": "Catalog Title",
  "size": 98765,
  "modified": "2026-03-24T10:00:00Z",
  "created": "2026-03-01T09:00:00Z",
  "hasHtml": true
}
```

- `size`: int64, file size in bytes — consumed by Browse tab size column
- `modified`: RFC3339 string — consumed by `new Date(value).toLocaleString()` in BrowseCatalogsTab line 150
- `created`: RFC3339 string — present as data field; not currently rendered in Browse tab column definitions (data-only fix, no UI change)

### HTML Tree Root Node (CATL-04)

Root node in generated HTML must render as:
```
└── [size]&nbsp;&nbsp;.<br>
```

Where `[size]` is the human-readable byte format (e.g., `1.2 MB`) and `.` is
`basename("./")`. This matches Electron's `generateTreeStructure` output with
`isLast=true` and `prefix=""`.

## Copywriting Contract

| Element | Copy | Context |
|---------|------|---------|
| Create success notification | "Catalog created successfully — [N] files, [size]" | Ant Design message.success() after CreateCatalog returns; fileCount and totalSize from CATL-02 result |
| Create success: file location | "Saved to: [jsonPath]" | Displayed below success notification or in result detail |
| Browse empty state heading | "No catalogs found" | When BrowseCatalogs returns empty array |
| Browse empty state body | "Select a directory to load catalogs, or create a new catalog first." | Sub-text below heading in empty Browse tab |
| Traversal error (skipped item) | Silent — logged to console.warn only | Inaccessible files silently skipped; no user-visible message |
| Symlink traversal (CATL-03) | No user-visible copy change | Symlinks now counted; reflected in fileCount only |

Destructive actions in Phase 1: none. Phase 1 has no delete, overwrite confirmation,
or destructive operations.

## Interaction States Affected by Phase 1

No new interaction states are introduced. The following existing states receive
corrected data after Phase 1 completes:

| State | Tab | Before Fix | After Fix |
|-------|-----|-----------|-----------|
| Create catalog success | Create | result is undefined (no return value) | result contains fileCount, totalSize, paths |
| Empty directory in tree | CatalogModal (HTML view) | contents may be null → TypeError | contents is always [] |
| Root node in HTML tree | CatalogModal (HTML view) | `./<br>` — no connector, no size | `└── [size]&nbsp;&nbsp;.<br>` |
| Browse catalog size column | Browse | undefined or missing | byte value from DATA-03 |
| Browse modified column | Browse | may fail new Date() parse | RFC3339 string — always parseable |
| Symlinked files in catalog | Create | silently skipped | counted and traversed |
