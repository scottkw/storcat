# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Scope Note

Phase 3 is primarily a Go backend phase (config struct extension + lifecycle hooks) with one frontend change: replacing two hardcoded stubs in `wailsAPI.ts` with real calls to new Go methods. No new UI components, screens, or layouts are introduced. The Settings modal that surfaces the "Remember Window Size & Position" toggle already exists in `MainContent.tsx` — this phase makes it functional. The design contract below documents the existing visual context and the interaction contract for the one toggle this phase wires up.

## Interaction Contract

The one interaction this phase makes functional:

### Window Persistence Toggle (Settings Modal > General section)

| State | Component | Behavior |
|-------|-----------|----------|
| Toggle ON (default) | `<Switch checked={true} />` | Ant Design Switch in checked/blue state. On mount, reads `GetWindowPersistence()` from Go config. Default: `true` (matches `DefaultConfig()` in Go). |
| Toggle OFF | `<Switch checked={false} />` | Ant Design Switch in unchecked/grey state. Calls `SetWindowPersistence(false)` on Go config. Config file on disk updates immediately. |
| Toggle change (optimistic) | Switch changes state before IPC resolves | `setWindowPersistence(enabled)` is called in state first, then `window.electronAPI.setWindowPersistence(enabled)` is awaited. No loading indicator — IPC call is fast (<50ms). |
| Load error | Falls back to `true` | `console.warn('Failed to load window persistence setting:', error)`. No visible error state — toggle defaults to ON. |
| Save error | `console.error` only | No toast, no rollback of toggle state. Existing pattern retained. |

No new interaction states, loading spinners, or error toasts are introduced in Phase 3. The wailsAPI wrapper handles all error suppression internally.
