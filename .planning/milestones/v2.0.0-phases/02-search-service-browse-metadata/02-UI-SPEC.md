Source: `frontend/src/index.css` CSS custom properties (light theme defaults). Colors are runtime-overridable via 11 built-in themes in `themes.ts`. All new UI must use CSS variables, never hardcoded hex values.

| Role | CSS Variable | Default Value | Usage |

| Dominant (60%) | `var(--app-bg)` | `#f8f9fa` | Page background, content area background |
| Secondary (30%) | `var(--card-bg)` / `var(--sidebar-bg)` | `#ffffff` | Cards, table rows (even), modal background, sidebar |
| Accent (10%) | Ant Design `type="primary"` (default `#1890ff`) | `#1890ff` | Primary action buttons only (see accent list below) |

**Secondary surfaces used in Browse tab:**
- `var(--table-stripe)` (`#f8f9fa`) — alternating table rows (odd), sticky table header, pagination bar background
- `var(--table-hover)` (`#e9ecef`) — row hover state
- `var(--border-color)` (`#dee2e6`) — table borders, cell borders, container borders

### Size Column Specification

Add to `BrowseCatalogsTab.tsx` columns array, positioned between `name` (Catalog File) and `modified` (Modified):

```typescript
{
  key: 'size',
  title: 'Size',
  width: 120,
  minWidth: 80,
  render: (value: number) => {
    if (value === 0) return '0 B';
    const units = ['B', 'KB', 'MB', 'GB'];
    const i = Math.floor(Math.log(value) / Math.log(1024));
    return `${(value / Math.pow(1024, i)).toFixed(1)} ${units[i]}`;
  },
},
```

Source: `02-RESEARCH.md` — Pitfall 4 and Code Examples section. Column order mirrors typical file manager convention and matches `CatalogMetadata` field order.

**Size column rendering rules:**
- `0` renders as `"0 B"` (not blank, not `"0.0 B"`)
- Bytes under 1024 render with no decimal: handled by `toFixed(1)` — output is e.g. `"512.0 B"`. Acceptable.
- Values render right-aligned if the executor chooses to add `textAlign: 'right'` style — recommended but not required.
- Cell uses inherited `font-family: monospace` and `font-size: 0.85rem` from `ModernTable` `td` styles. Do not override.

## Interaction Contract

### Browse Tab — Existing Interactions (unchanged)

| Interaction | Trigger | Outcome |
|-------------|---------|---------|
| Select Catalog Directory | Click primary button | Native OS folder picker opens; selected path stored in state + localStorage |
| Load Catalogs | Click primary button | Calls `getCatalogFiles`; populates table; shows `message.success` with count |
| Sort column | Click column header | Ascending → descending → unsorted cycle |
| Filter column | Type in column filter input | Filters table rows in real time; resets to page 1 |
| Resize column | Drag resize handle | Column width adjusts live |
| Open catalog | Click `.catalog-link` in Title column | Dispatches `openCatalogModal` custom event |

### Browse Tab — New Interactions (Phase 2)

| Interaction | Trigger | Outcome |
|-------------|---------|---------|
| Size column display | `browseCatalogs` result contains `size` field | Size column shows human-readable value (e.g., "1.2 MB") |
| Size column sort | Click "Size" header | Numeric sort on raw `size` value (not formatted string) |
| Size column filter | Type in Size column filter input | Filters on formatted value string — acceptable for this phase |

**Loading state:** Existing `state.isLoading` spinner via `ModernTable` `loading` prop. No change required.

**Error state:** Existing `message.error()` pattern via Ant Design notification. No change required for error display.

## States

All states apply to `BrowseCatalogsTab`. Executor must handle all four.

| State | Condition | UI Behavior |
|-------|-----------|-------------|
| Idle (no directory) | `!state.selectedBrowseDirectory` | "Load Catalogs" button disabled; empty state shown in table area |
| Loading | `state.isLoading === true` | "Load Catalogs" button shows spinner; `ModernTable` shows "Loading..." centered in container |
